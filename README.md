# CS-300 Portfolio
Analysis of the Run Time and Memory of Data Structures

For this course, I worked mainly on two projects that focused on improving on and handling course data efficiently.

The first project involved analyzing different data structures, vectors, hash tables, and binary search trees, to

determine their runtime and memory performance when storing course information. I wrote pseudocode to load course

data from a file, check prerequisites, and search or print course details. This helped me see how each structure’s 

strengths and weaknesses affected the program, like vectors being simple and fast. Hash tables being great for things

like search but can end up complicating sorting, and binary search trees having a balance between search speed and

ordered printing.


The second project combined all of these ideas into a working C++ program that reads a course file, stores courses

in a binary search tree, and allows a user to print the full course list or view individual courses with their 

prerequisites. This project gave me hands-on experience implementing a data structure, parsing files, handling 

input, and traversing a tree. One of the challenges was ensuring the CSV parsing handled edge cases like missing 

prerequisites. Through this, I learned the importance of testing, modular design, and writing good code that can

handle unexpected input gracefully.


Overall, these projects strengthened my understanding of data structures, algorithms, and programming skills.

They pushed me to think about how data is stored, accessed, and displayed, and how design decisions affect

usability and efficiency. By completing these projects and reflecting on my process, I gained a fair amount 

confidence in building programs that are easy to follow, readable, and functional. This experience will hopefully

be something that will carry forward into future courses and professional development.
